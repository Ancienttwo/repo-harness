## Why

The minimal/full protocol migration is merged on main and is the intended
0.11.1 patch. The first protected ship attempt also reproduced a macOS Bash
3.2 failure: expanding an empty `child_args` array under `set -u` aborts
primary-to-linked-worktree delegation before finish, merge-gate, push, and PR.
Acceptance preflight then exposed a second release blocker in newly adopted
repos: evidence-ledger writes are neither ignored nor excluded from the review
subject, so successful emission makes its own subject stale. Publishing without
both repairs would ship broken default closeout and acceptance paths.

## Goal

Publish `repo-harness@0.11.1` from an exact verified main commit containing the
minimal/full migration, the bounded Bash 3.2 ship repair, and the evidence-ledger
subject-stability repair, then reinstall the registry package and migrate the
operator runtime to protocol-2 `full` with 11/11 Codex hooks and a healthy agent
fleet.

## Scope

- In scope:
  - make empty optional child arguments safe under `/bin/bash` while preserving
    non-empty `--dry-run` and `--ready` forwarding;
  - keep source and packaged ship helpers byte-identical and add a focused host
    shell regression guard;
  - classify `.ai/harness/evidence/` as ignored runtime state and an operational
    review path in both adoption authorities, with manifest and regression
    coverage;
  - synchronize all 0.11.1 version, changelog, README, release-filing, and
    current-status projections;
  - run targeted, root, contract, and full release verification;
  - merge to main, publish npm, tag, create the GitHub Release, reinstall from
    npm, explicitly migrate installed state to `full`, refresh adapters, and
    prove final local/external truth.
- Out of scope:
  - changing agent role effort, adding profile aliases, changing workflow profile vocabulary, or absorbing unrelated branches/worktrees.
- Taste constraints: preserve one ship command contract; no alternate closeout
  path, compatibility alias, or best-effort fallback.

## Stop Conditions

- Stop and hand back to the parent if the change would require editing a path outside Allowed Paths.
- Stop if an Exit Criteria command cannot be run in this environment.
- Stop if Goal, Scope, or Exit Criteria are internally contradictory.
- Stop before publish on any release-gate failure, unexpected dirty path, moved
  main, missing npm authority, or stale acceptance evidence.
- Stop after publish if registry truth is uncertain; do not retry blindly
  against immutable npm version state.

## Falsifier

If the zero-argument expansion still fails under `/bin/bash -u`, or forwarding
one optional argument changes its byte value, the repair is wrong. The cheapest
proof is a focused `/bin/bash` argument-count/value probe before the full gate.

## Root Cause Evidence

Required when Task Profile is `bugfix`; leave as-is otherwise.

- root_cause: one sentence naming file:line/condition (testable, not "a state issue").
- repro: the command or UI path that reproduces the symptom.
- regression_guard: path to a test that fails on the unfixed code and passes after the fix (must also appear under exit_criteria.tests_pass).
- pre_fix_failure_artifact: path to a captured run of regression_guard on the UNFIXED code. Capture with `bun test <regression_guard> > <artifact> 2>&1; echo "PRE_FIX_EXIT=$?" >> <artifact>` (no pipes — pipes swallow the exit status). The gate requires a non-zero `PRE_FIX_EXIT=` line plus the regression_guard path string in the artifact (see the Root Cause Evidence Gate section in docs/reference-configs/sprint-contracts.md).
