## Design Decisions

- Keep the existing child-helper command contract and use Bash's
  `${array[@]+"${array[@]}"}` expansion so an unset-length array becomes zero
  arguments under macOS Bash 3.2 while populated arrays retain argument
  boundaries.
- Publish the already-merged minimal/full profile migration and this
  release-blocking ship repair together as `0.11.1`; no profile aliases,
  fallback parser, or alternate closeout route is introduced.
- Treat the evidence ledger as runtime authority, not semantic diff: both
  adoption-time ignore projection and review-subject normalization exclude
  `.ai/harness/evidence/`. Either defense alone is insufficient for already
  adopted repos or fresh repos.
- Release disclosure is computed from the complete `v0.11.0..candidate` range,
  not only from the final profile and ship-fix work-package. Post-freeze gate
  hashes live in the operational review/evidence surfaces so the semantic
  filing does not create another evidence self-reference.

## Tradeoffs Considered

| Option | Decision | Reason |
|--------|----------|--------|
| Conditional array expansion | Use | It is supported by Bash 3.2 and preserves both the zero-argument and multi-argument contracts under `set -u`. |
| Disable nounset or append a sentinel | Reject | Either weakens the helper invariant or changes the child command's observable arguments. |
| Ignore plus subject exclusion | Use | The generated ignore prevents new runtime residue while normalization keeps existing or manually surfaced ledgers from changing semantic acceptance. |
