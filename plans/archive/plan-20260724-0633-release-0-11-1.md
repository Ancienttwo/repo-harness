## Captured Planning Output

## Why

The minimal/full protocol migration is merged on main and is the intended 0.11.1 patch. The first protected ship attempt also reproduced a macOS Bash 3.2 failure: expanding an empty child_args array under set -u aborts primary-to-linked-worktree delegation before the finish, merge-gate, push, and PR steps. Release 0.11.1 must publish the migration and the smallest coherent repair for that release-blocking helper defect.

## P1: Architecture Map

- Release version authority: package.json, assets/skill-version.json, .claude/.skill-version, localized README release projections, docs/CHANGELOG.md, and the dated deploy release filing.
- Ship runtime authority: scripts/ship-worktrees.sh with byte-identical packaged projection assets/templates/helpers/ship-worktrees.sh.
- Regression surface: tests/unit/closeout-runner-guardrails.test.ts plus source/package parity tests.
- Release gates: bun run check:release before publication and scripts/check-release-published.sh after npm/tag/GitHub release.
- Operator runtime: Bun global repo-harness install, explicit protocol-1 to full migration, host adapter refresh, and setup/status readback.
- Out of scope: changing agent role effort, adding profile aliases, changing workflow profile vocabulary, or absorbing unrelated branches/worktrees.

## P2: Concrete Trace

1. Patch the two authoritative ship helper copies so an empty optional-argument array expands to zero arguments under /bin/bash 3.2, while non-empty --dry-run/--ready forwarding is unchanged.
2. Add a host-shell regression guard and keep source/package helpers byte-identical.
3. Bump all 0.11.1 version projections, changelog, and release filing as one release commit.
4. Run targeted checks and the exact full release gate, close the contract, merge to main, and confirm main/origin parity.
5. Publish the exact main commit to npm, read back the registry, create/push annotated v0.11.1, create the GitHub Release, and run the published-release verifier.
6. Reinstall repo-harness@0.11.1 from npm, explicitly migrate legacy installed profile state to full, refresh both adapters, and verify protocol 2, 11/11 Codex hooks, agent fleet, PATH version, and registry source.

## P3: Design Decision

Use one patch release containing the already-merged profile migration and one bounded Bash invocation repair. Do not add another agent role or compatibility alias. The shell change preserves the existing command contract and only makes the zero-optional-argument case representable on the supported macOS system Bash. If the full release gate or post-publish readback fails, stop at that boundary rather than publishing or claiming success.

## Scope

- Repair empty child argument forwarding in source and packaged ship helpers.
- Add a focused Bash 3.2 regression guard.
- Synchronize all 0.11.1 version, changelog, README, task snapshot, and release filing surfaces.
- Verify, merge, publish npm, tag, create GitHub Release, reinstall, migrate to full, refresh adapters, and prove final local/external state.
